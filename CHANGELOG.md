# Changelog

All notable changes to this project will be documented in this file.

## 1.3.1

- Ajusted some colours 
- Updated images to be real.
- Added `made-by-eura` tag to VSCode marketplace tags
- renamed theme modes

## 1.3.0

- Added semantic token support for modern language servers.
- Expanded Git, merge, minimap, overview ruler, and diagnostics color coverage.
- Improved language-oriented scope support for JSON, Markdown, TypeScript, JavaScript, Rust, Python, Lua, Swift, CSS, HTML, C++, C#, Kotlin, Java, and Tauri-oriented config files.
- Cleaned extension metadata and added marketplace assets/documentation polish.

## 1.2.4

- Switched icon system color from Indigo to Cyan (#00C0E8).
- Updated activity bar icon foreground to Cyan for consistency.

## 1.2.3

- Set icon foreground system to Indigo (#6155F5).
- Lightened Git add/delete colors for reduced visual harshness.
- Set both warning and error diagnostics/icons to Orange (#FF8D28).

## 1.2.2

- Normalized primary accent usage to Blue (#0088FF) for both light and dark themes.
- Replaced remaining indigo/cyan accent roles in core UI states and key token accents.

## 1.2.1

- Switched main accent color to Blue (#0088FF) across light and dark themes.
- Replaced teal/cyan accent usage in UI highlights, borders, and key syntax accents.

## 1.1.0

- Added Cupertino Dark theme variant.
- Updated extension metadata to include both light and dark themes.

## 1.2.0

- Softened both light and dark palettes to reduce visual strain.
- Rebalanced editor accents, selections, and status bars for lower intensity.
- Added troubleshooting note for dark theme visibility after VSIX updates.

## 1.0.0

- Initial release of Cupertino.
